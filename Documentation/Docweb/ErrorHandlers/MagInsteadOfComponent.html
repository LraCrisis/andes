Using the magnitude instead of the component of a vector

(as of 3/26/01)

This has to be tested after more specific bugs, including MissingNegOnVectorMag.

If the student's entry is an equation

  • that contains (mag ?vector)
  • and ?axis is an axis that ?vector can be projected along
  • and replacing (mag ?vector) with (compo ?vector ?axis) makes the student's equation correct
then the hint sequence is
  • preamble
  • (hint string (point) "~a is best stated with all the quantities expressed in component form." law for the psm)
  • (hint string (point) "Have you used the magnitude of a vector where you should use the vector's component along an axis?")
  • (hint string (bottom-out) ...)

a5 Eq-Pitt 48-43

On the ring pulled by 3 forces, the student has drawn a FBD without acceleration, entered givens, entered projection equations for all 3 forces, and written "FS3=Ft1_x+Ft2_x+Ft3_x."  This would be correct if (a) FS3 were defined as the net force and (b) its x-component were used here instead of its magnitude.  Let's assume for the sake of argument that the acceleration and the net force are non-zero, and that they are not along the x-axis.
  • T1 asks what is the sum of forces.
  • T2 asks the student to define a variable for the net force, then to use its x-component in this equation.
  • Andes says that the FS3 is not defined.
  • KVL: Andes should first say that FS3 is not defined, then it should tutor the student on the sum-of-forces principle.
Andes first gives an undefined variable message, so let's assume that the student defines FS as the net force, reenters the equation, gets red, and asks for help.  Because FS is non-zero and FS_x is not equal to FS, the equation is incorrect.  The intended equation is FS_x=Ft1_x+Ft2_x+Ft3_x, which is primitive and non-premature, so the error handler gives the hint sequence above.
