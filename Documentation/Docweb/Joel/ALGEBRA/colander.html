Problem solver for Andes2

Joel Shapiro

The beginning of the use of the algebra system in the history of a physics problem is to solve the “canonical” equations. This can be done by the stand-alone program colander, which gets its input from a probname.eqf file, or from the help system with the Solver.dll. The general procedure in either case is outlined in “Canonical Equation Solving”. Here we describe the functions involved in colander.

The program is called with possible debug flags, and if debugging is used it will produce a lot of output, so output should be redirected to a file. Thus the calls might be

./colanderif not debugging
./colander hexflags   >   outfile if debugging
where hexflags is a hex number giving the diagnostic level, also described in dbg.h.

The main of the colander program is in the file colget.cpp. It sets up the path for finding the input file, which it currently expects to find in the directory C:/joel/math/data/eqs.wur/. It prompts for the name of the problem, probname, which must be given (case-sensitively ?) without extension. Colander tacks on .eqf, and reads the file from the above directory. It will write the solutions in a file in the current directory, called probname.jsol.

Colget initializes the table of unit abbreviations, calls getallfile to read in the input information, sets an ofstream to the output file, and calls solveeqs If solveeqs finds no problems, it then calls checksol on each equation and reports any discrepancies in the evaluation. Then it calls dimchkeqf to check for discrepancies in the units.

solveeqs makes a copy of all the input equations, does elementary simplifications and dimensional analysis checking, adds the assignment statements for parameters, and then calls checkeqs to actually solve the system of equations. If checkeqs fails to use up all equations or solve for all variables, this is then reported in the output file.

Most of the work is done in checkeqs The file checkeqs.cpp has extensive comments at the beginning describing its approach, though much of this described in Canonical Equation Solving.
