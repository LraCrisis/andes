Details of the dnum specification

Joel Shapiro

Physical quantities with units need to have their units specified whenever they are given a numerical value. In equations, this is done with a DNUM, in the form
(= varname (DNUM number unitstring)
where varname is the name of the variable, number is a numerical value, and unitstring describes the units as specified below.

When a variable is declared, its units are given in the form of a string of "preferred units", which may be standard SI units or other known units, in the form

(SVAR varname unitstring)

The Unit String

The unitstring is composed of units, possibly multiplied together with . (preferred) or * (non-standard), divided with /, and raised to explicit numerical value with ^ num. No parentheses are allowed. The units consist of a base unit and possibly a modifiying prefix. The base units are given in the file units.h, and include all SI units, m, kg, s, C, K, g, N, J, V, A, T, Wb, ohm, Hz, Pa, F, H, W, and a number of nonstandard ones, including deg, rad, rev, lb, day, hr or h, min, yr, liter, ft, in, mi, eV, dyne, erg, cal. Note that none include a period, as we cannot destinguish one from the center-dot which is the standard symbol for multiplying units.
The prefixes are most of the SI ones, including a, f, p, n, mu (or $m), m, c, d, k (or K), M, G, T, P, and E.

More information about how the preferred units interact with the standard units is in the variable specification.
