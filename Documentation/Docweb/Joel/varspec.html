Details of the variable specification

Joel Shapiro

Physical variables are given `canonical' names by the SGG system, which also determines the physical units in which to express this variable. This information is sent to the algebra system in a declaration, or SVAR statement. At problem-solving time, the form of the statement to the algebra system is

(SVAR varname unitstring )
After the problem has been solved, the algebra package returns the solution for each variable with a statement
(SVAR varname number unitstring )
The variable name varname is composed of (case-sensitive) letters, numbers, _, and $, beginning with a letter or $. It must not have spaces, -, ., or anything else.

The number is an integer, floating point expression, or $P, which represents pi. The floating point expression may have an exponent in usual C/Fortran form.

The unitstring is composed of units listed in units.h, possibly coupled with prefix modifiers as described in prefixes.h, and combined with . for multiplication, / for division, and ^number for powers. No parentheses are allowed.

When the help system wants the algebra system to help in bubble growing or for tutoring help, the form it uses to declare the variables is

(indyAddVar varname number unitstring)

Variable attributes

The solver needs to know if there are conditions on the values of the variables to be solved. The help system specifies these conditions with these statements:
  • (nonnegative varname)   The variable cannot take a negative value
  • (positive varname)   The variable can only have values larger than zero
  • (nonzero varname)   The variable cannot have a zero value
  • (parameter varname)   The variable's value cannot be determined from the equations. The solution to the problem is supposed to work for any positive value of the variable, with values of other variables possibly expressed algebraically in terms of the parameters.
