Do we need buggy operators in the KB?

(as of 3/30/01)

Kurt:

It is traditional to analyze errors by using buggy operators.  A buggy operator is just a regular operator except that its conclusions are false.  If one uses buggy operators, then all propostoins have to be marked to distinguish correct ones from those produced by bugs.

The problem with buggy operators is that they can combine combinatorially with correct operators and produce a horendous number of solutions.

Fortunately, all errors I have seen so far can be interpreted by modifing the students entry to make it correct (see discussion).  Thus, it appears that we do not (yet) need buggy operators.  Let's design under the assumption that buggy ops will not be needed.
