why

why
Description
Gets an explanation of the last hint it gave, based on the rule that is identified with the hint in the solution graph. The explanations are canned text that are stored in the rules.lsp kb file.
Example
  "(why)"
Returns
A hint descriptor return value.
Arguments
none.
Side Effects
Updates the Assessor bayesian network to reflect the fact that the student has received this hint.
