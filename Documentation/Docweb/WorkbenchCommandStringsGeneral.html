Workbench Command Strings: General

The workbench exposes a command string interface to other programs. The ANDES log player uses this interface when it replays the session log: an ANDES log file contains a sequence of commands sufficient to recreate all the student actions on the interface. The Workbench command string interface also supplies the means by which the help system can send commands to display help messages on the workbench interface.

A workbench command string consists of the command identifier followed by a space followed by a command-specific list of arguments. In most cases the arguments are alphanumeric tokens delimited by white space. However, some commands like “show-hint” make use of “trailing string” arguments, in which everything from the start of the argument position to the end of the string is taken as the argument. Trailing string arguments can therefore include spaces.

The Workbench commands that are primarily intended for use by the help system are:

show-hint <hint descriptor>     show a hint in the tutor msg pane
show-lesson lesson-file-name    show an ANDES mini-lesson
open-browser URL                open lesson viewer on any URL
close-browser                   close lesson viewer if open
msg <message text>              show text in a dialog box
training-card card-id           pop up a training card
show-demo demo-file-name        play a log file as a demo script

These commands are most often sent "piggy-backed" onto the return value of a help system API call.

It is also possible for the help system to send a command to the workbench asynchronously at any time. However, this facility is deprecated for the purpose of giving help since it involves unsolicited interruptions of the user’s work that could be annoying to the student.
